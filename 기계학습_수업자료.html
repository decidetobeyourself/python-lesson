전개 2 · 비교

전통적 프로그래밍 vs 기계학습

178쪽

스팸 메일 분류를 예로 두 방식을 비교해 봅시다.

📋 전통적 프로그래밍

사람이 직접 규칙을 정함.
"제목에 '이벤트'가 있으면 스팸" 처럼 사람이 규칙을 만듦.

입력+사람이 만든 규칙→정답

⚠ 새 상황마다 규칙을 계속 바꿔야 함

VS

🤖 기계학습

컴퓨터가 데이터를 학습해 규칙을 스스로 찾음.
수많은 메일에서 '당첨·무료·이벤트' 패턴을 자동 발견.

입력+정답(레이블)→규칙 학습

✔ 복잡한 문제를 효율적으로 해결

* 레이블(label) : 각 입력 데이터에 대해 원하는 출력값(정답)

전개 3 · 유형

기계학습의 세 가지 유형

179쪽

학습하는 방식에 따라 세 가지로 나뉩니다.

지도
🏷️

지도학습

입력 데이터 + 정답(레이블)을 함께 학습해 새 데이터를 예측·분류

예 : 강아지/고양이 사진 분류
비지도
🧩

비지도학습

레이블 없이 데이터의 숨은 패턴을 찾아 비슷한 것끼리 묶음(군집)

예 : 고객 구매패턴 그룹화
강화
🎮

강화학습

변화하는 환경에서 보상(reward)이 최대가 되는 행동을 시행착오로 학습

예 : 게임 AI · 로봇 제어

레이블이 있나? → 있으면 지도, 없으면 비지도.  |  보상으로 배우나? → 강화학습!
